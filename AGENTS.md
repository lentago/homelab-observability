## Quick facts

- **Logs:** Firewalla Promtail → Alloy `loki.source.api` on **:3100** → Grafana Cloud Loki (`GRAFANA_CLOUD_LOGS_*`).
- **Metrics:** Alloy blackbox + node + HA scrapes → Grafana Cloud Mimir (`GRAFANA_CLOUD_METRICS_*`).
- **Dashboards:** Edit `dashboards/*.json`, then `cd terraform && terraform plan && terraform apply`.
- **Secrets:** Never commit `.envrc`, `alloy/ha_token`, or `terraform/*.tfstate`.

## grafana-assistant CLI (optional)

`grafana-assistant` targets Grafana Cloud at `https://pitzilabs.grafana.net` (see `grafana-assistant config list`). Two distinct auth paths — don't confuse them:

- **Grafana HTTP API + Terraform:** the `GRAFANA_SA_TOKEN` in `.envrc` is sufficient. This is what `terraform/` and `scripts/inventory-cloud.sh` use.
- **Assistant A2A (`grafana-assistant prompt` / `chat`):** even an Admin-role SA token returns `HTTP 401: invalid user` against the A2A endpoint, and `grafana-assistant config list` shows `AUTH: Not authenticated` for the configured instance — A2A uses its own user-bound auth, separate from the SA token. Run `grafana-assistant auth` once (browser PKCE; **Assistant CLI User** role, which Editor or above gets automatically) before using `prompt` or `chat`. Until that's done, only local subcommands like `agents-md` and `config` work.

## grafana-assistant

Query metrics, investigate alerts, analyze logs/traces, and interact with
Grafana from scripts or the terminal via the A2A protocol.

Run `grafana-assistant --help` or `grafana-assistant <command> --help` to
discover commands and flags.

### Setup

```bash
grafana-assistant config set-instance prod --url https://your-stack.grafana.net
grafana-assistant auth                    # browser login (recommended)
grafana-assistant config use-instance prod
```

List instances: `grafana-assistant config list`

### Querying (non-interactive)

Always use `--json` for machine-readable output. Use `--continue` to build
on the previous conversation without repeating context.

```bash
grafana-assistant prompt "your question" --json
grafana-assistant prompt "follow-up" --continue --json
```

Drop `--continue` when switching to an unrelated topic.

| Flag | Purpose |
|------|---------|
| `--json` | Structured JSON output |
| `--continue` | Continue previous conversation |
| `--context <id>` | Resume a specific conversation by `contextId` |
| `--timeout <s>` | Increase timeout for complex queries (default 300) |
| `--instance <name>` | Target a specific instance |

### Example queries

```bash
grafana-assistant prompt "List all firing alerts in namespace backend-prod" --json
grafana-assistant prompt "Using prometheus, show CPU usage for namespace my-app over the last hour" --json
grafana-assistant prompt "Show error logs from namespace api-gateway in the last 15 minutes" --json
grafana-assistant prompt "Find dashboards related to application performance" --json
```

### Common mistakes

- **Missing datasource name** — queries without `"using <datasource>"` may hit the wrong datasource.
- **Vague time ranges** — always specify "last 1h", "last 15m", etc.
- **Long conversation chains** — drop `--continue` when switching topics; stale context confuses the assistant.
- **Forgetting `--json`** — without it, output is plain text that is harder to parse programmatically.
- **Graphs** — panel references like `[panel:p1]` only render in the Grafana UI, not in CLI output.

### Local tool access

```bash
grafana-assistant config add-project <project-name> <path>
grafana-assistant tunnel connect
```
